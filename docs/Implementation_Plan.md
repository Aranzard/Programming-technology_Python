### Что считается FULL режимом

**Режим `full`** выполняет полную перезагрузку всех данных:

- **Extract**: скачивает данные за последние 30 дней с USGS API
- **Transform**: полностью пересоздаёт normalized и mart слои
- **Load**: заменяет всю таблицу в PostgreSQL (DROP + CREATE + INSERT)
- **Watermark**: обновляется до максимальной даты в загруженных данных

**Что пересоздаётся:**
- `data/raw/` — новый JSON файл
- `data/normalized/` — новый CSV файл
- `data/mart/` — новый CSV файл
- Таблица `mart_earthquake_jp` в PostgreSQL (полная замена)

**Идемпотентность:** повторный запуск `full` даёт идентичный результат.

### Что считается INCREMENTAL режимом

**Режим `incremental`** загружает только новые данные:

- **Какие данные считаются новыми:** все события с `time` > `watermark`
- **Как используется watermark:** API запрашивает данные с `starttime = watermark`
- **Защита от дублей:** upsert по business key `(date, region_id)`

**Логика работы:**
1. Читаем `watermark` из `state.json`
2. Запрашиваем в API все события после этой даты
3. Нормализуем новые данные
4. Объединяем с существующей витриной
5. Загружаем через upsert (удаляем старые записи за те же даты, вставляем новые)
