# Data Quality Contract (Data Contract)

## Общая информация

| Параметр | Значение |
|----------|----------|
| **Вариант** | 16 |
| **Тема** | Землетрясения (USGS) - Япония (Хонсю) |
| **Слои проверки** | normalized, mart |

---

## DQ-правила для слоя NORMALIZED

| № | Правило | Тип | Критичность | Описание |
|---|---------|-----|-------------|----------|
| 1 | `normalized_not_empty` | FAIL | Жёсткая | Таблица normalized не должна быть пустой |
| 2 | `event_id_not_null` | FAIL | Жёсткая | event_id не может быть NULL |
| 3 | `event_id_unique` | FAIL | Жёсткая | event_id должен быть уникальным |
| 4 | `magnitude_range` | WARNING | Мягкая | Магнитуда в диапазоне [0, 10] |
| 5 | `depth_positive` | WARNING | Мягкая | Глубина >= 0 км |

---

## DQ-правила для слоя MART

| № | Правило | Тип | Критичность | Описание |
|---|---------|-----|-------------|----------|
| 6 | `mart_not_empty` | FAIL | Жёсткая | Витрина не должна быть пустой |
| 7 | `business_key_unique` | FAIL | Жёсткая | Бизнес-ключ (date, region_id) уникален |
| 8 | `mart_no_null` | FAIL | Жёсткая | Нет NULL в ключевых полях (date, cnt_events, region_id) |
| 9 | `positive_events` | WARNING | Мягкая | cnt_events > 0 |

---

## Бизнес-ключ витрины

**Бизнес-ключ:** `(date, region_id)`

**Почему:** Каждая запись в витрине уникальна для конкретной даты и региона.

---

## Что происходит при нарушении

| Критичность | Действие |
|-------------|----------|
| **FAIL (жёсткая)** | Pipeline останавливается, данные не загружаются |
| **WARNING (мягкая)** | Pipeline продолжает работу, но в отчёт добавляется предупреждение |

---

## Запуск DQ-проверок

```bash
# Запуск всех проверок
python -m src.dq

# Запуск только normalized
python -m src.dq --layer normalized

# Запуск только mart
python -m src.dq --layer mart
