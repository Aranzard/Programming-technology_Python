# Data Dictionary — mart_earthquake_jp
**Слой:** mart (агрегированная витрина)  
**Источник:** USGS Earthquake API → normalized → mart

---

## Назначение словаря

Словарь данных поясняет **бизнес-смысл** каждой колонки витрины `mart_earthquake_jp`. Используется, чтобы быстро понять, что означает та или иная метрика без погружения в код.

---

## Колонки витрины

| column | business meaning | unit | notes / example |
|--------|------------------|------|-----------------|
| `date` | Календарная дата, за которую агрегированы события | YYYY-MM-DD | Агрегация в UTC. Пример: `2026-03-23` |
| `cnt_events` | Количество землетрясений, зарегистрированных в этот день (магнитуда ≥ 4.0) | событий / день | Пример: `5` означает, что за день произошло 5 землетрясений |
| `max_mag_day` | Самая сильная магнитуда среди всех событий дня | магнитуда  | Пример: `6.5` — землетрясение магнитудой 6.5 |
| `avg_mag` | Средняя магнитуда всех событий дня. Показывает "типичную" силу землетрясений в этот день | магнитуда  | Пример: `4.6` — в среднем землетрясения были магнитудой 4.6 |
| `min_mag_day` | Самая слабая магнитуда среди всех событий дня (минимальное значение) | магнитуда  | Пример: `4.2` — самое слабое землетрясение дня |
| `deep_events_percent` | Доля событий, произошедших на глубине более 70 км. Глубокие события обычно менее опасны для поверхности | проценты (%) | Пример: `23.4%` означает, что 23.4% всех событий за период были глубокими |
| `total_events` | Общее количество землетрясений за весь анализируемый период. Константа для всех строк витрины | событий | Пример: `47` — всего 47 землетрясений за период |
| `max_magnitude_overall` | Максимальная магнитуда за весь анализируемый период. Константа для всех строк витрины | магнитуда  | Пример: `6.5` — самое сильное землетрясение за весь период |
| `region_id` | Идентификатор географического региона | — | Всегда `JP_HON` (Япония, Хонсю) |
| `region_name` | Человекочитаемое название региона на русском | — | Всегда `"Япония (Хонсю)"` |
